# Map Studio — 지도 시각화 & PPT 익스포트 기능 계획

> 작성일: 2026-06-08 · 분양 기획 실무의 "입지 지도 제작" 자동화 기능 설계 문서
> 향후 세션이 이 문서만 읽어도 배경·목표·범위·단계를 파악할 수 있도록 작성.

---

## 0. 한 줄 요약

분양 기획 보고서의 **입지 분석 지도**를, 손으로 선을 따라 그리는 방식에서
**공개 지리 데이터를 레이어로 얹는 방식**으로 바꾸고, 결과물을 **편집 가능한 PPT**로 내보낸다.

---

## 1. 배경 — 실무의 통증 (Why)

현재 분양대행사 기획 실무의 입지 지도 제작 방식:

1. 아무 정보 없는 **깨끗한 지도를 캡처**해 PowerPoint에 삽입
2. 네이버지도·호갱노노 등 **정보가 있는 지도를 그 위에 겹쳐** 놓고
3. 지하철 노선·도로·학교·병원·산업단지 구획을 **사람이 눈으로 보고 선을 따라 그림**
4. PPT로 만드는 이유 = **향후 수정 가능성** (발표 직전까지 선·구획을 손봄)

### 핵심 비효율
- **곡선** 철도·도로, **산업단지 구획 폴리곤**을 수작업 트레이싱 → 시간 과다
- 이미 디지털로 존재하는 선형을 사람이 다시 그리는 중복 노동

### 발상 전환
> 트레이싱이 아니라 **데이터 렌더링**. 곡선은 이미 데이터(LineString)에 담겨 있다.
> 호갱노노가 선을 그려주는 것과 동일 원리 — 우리는 공개 데이터를 우리 스타일로 그린다.

## 2. 목표 (What)

| 목표 | 설명 |
|------|------|
| G1. 레이어 기반 지도 | VWorld/MapTiler 배경 + 공개 데이터(지하철/도로/학교/병원/산단) 레이어 토글 |
| G2. 반경 기반 추출 | 단지 좌표 중심 2/3/5km 버퍼로 해당 영역 데이터만 잘라 가볍게 렌더 |
| G3. 광역입지 모드 | 시·도 단위 넓은 범위 시각화 |
| G4. 직접 그리기 | 예정·착공 노선 등 데이터 없는 요소는 펜(베지어) 도구로 수기 추가 |
| G5. PPT 익스포트 | 배경=이미지, 핵심 선/구획=편집 가능 벡터 도형(하이브리드) |

## 3. 데이터 소스 (확인 완료)

### 점(Point) — 쉬움, 공식 데이터 그대로
| 레이어 | 출처 |
|--------|------|
| 지하철역 | 공공데이터포털 (서울교통공사 1~8호선 위경도 등) |
| 학교 | 학교알리미 / 공공데이터포털 |
| 병원 | 건강보험심사평가원 |
| 관공서·편의시설 | 공공데이터포털 / OSM POI |

### 선(LineString) — OSM이 정답
> 공공데이터포털 노선 데이터는 운영기관별로 파편화 + 깔끔한 선형 아님.
> 실제 **곡선 선형**을 통째로 가진 곳은 **OpenStreetMap**.

| 레이어 | OSM 태그 |
|--------|----------|
| 지하철·철도 | `railway=subway / rail / light_rail` |
| 도로(위계별) | `highway=motorway / trunk / primary / secondary` |

### 면(Polygon)
| 레이어 | 출처 |
|--------|------|
| 산업단지 경계 | 한국산업단지공단 / VWorld / OSM `landuse=industrial` |
| 지적도(필지) | VWorld 지적도 레이어 |
| 행정경계 | VWorld / OSM |

### 라이선스 주의 (중요)
- ✅ **VWorld** (국토부, 출처표기 시 자유), **MapTiler**, **OSM**(ODbL, 출처표기 시 재가공·문서삽입 자유)
- ⚠️ **네이버지도 / 카카오맵 API**: 약관상 캡처·재가공·외부 문서 삽입 **제한** → 참고용으로만, 산출물엔 사용 금지

## 4. 기술 스택

| 역할 | 라이브러리 | 비고 |
|------|-----------|------|
| 지도 렌더 | **MapLibre GL JS** (UMD) | 빌드 불필요, 스크립트 태그로 추가 가능 |
| 배경 타일 | VWorld / MapTiler | API 키 필요(무료 티어), **온라인 필요** |
| 지리 연산 | **turf.js** | 버퍼(반경 원), bbox 필터, 라인 단순화 |
| 데이터 추출 | **Overpass API** | OSM 영역 쿼리 (bbox 기반) |
| 라인 단순화 | Douglas-Peucker (turf `simplify`) | PPT 도형 경량화용 |
| PPT 생성 | **PptxGenJS** (UMD) | `custGeom` + `cubicBezTo`/`quadBezTo` 베지어 곡선 지원 확인됨 |

### 아키텍처 제약 (현재 프로젝트 원칙)
- **빌드 없음** → 모든 라이브러리 UMD 스크립트 태그로 로드
- **오프라인 동작 원칙과 충돌**: 지도 타일·Overpass는 인터넷 필요
  → 타일/쿼리는 온라인 전용으로 명시, 추출된 GeoJSON은 **로컬 캐시**(프로젝트에 저장)하여 재방문 시 오프라인 표시

## 5. PPT 익스포트 — 두 경로

검증 결과 둘 다 가능하나 난이도 차이 큼.

### 경로 A — 래스터(이미지) · 쉬움 (1~2일)
- 지도를 PNG로 렌더 → PptxGenJS `addImage`로 슬라이드 삽입
- 장점: 빠르고 안정적 / 단점: PPT에서 **선 수정 불가**

### 경로 B — 벡터(편집 가능 도형) · 난이도 높음
- 파이프라인: `GeoJSON → 투영 좌표 → SVG path → PPTX custGeom(베지어)`
- PptxGenJS `custGeom`이 `cubicBezTo`/`quadBezTo` 지원 → **곡선 철도·도로를 PPT 네이티브 자유형 도형**으로, 발표자가 점을 직접 수정 가능
- 난관:
  1. 좌표 투영 변환 (위경도 → 슬라이드 EMU 좌표)
  2. SVG arc ↔ PPTX arc 표현 차이 (x,y,rx,ry,... ↔ x,y,hR,wR,stAng,swAng)
  3. 점 많은 도로 → 도형 과중 → `turf.simplify`로 단순화 필수

### 권장: **하이브리드**
> 배경지도 = 경로 A(이미지 고정), 핵심 선/구획(지하철·도로·산단) = 경로 B(편집 가능 벡터).
> 실무의 "배경은 고정, 선만 손본다" 패턴과 정확히 일치.

## 6. 기능 명세 (Map Studio)

### 6.1 지도 뷰
- 단지 핀 + 반경 원(2/3/5km 동심원, 토글)
- 좌측 레이어 패널: 지하철 / 철도 / 도로(위계별) / 학교 / 병원 / 산단 / 지적도 토글
- 상단 모드 전환: **근린입지(반경)** ↔ **광역입지(시·도)**

### 6.2 데이터 로딩
- "이 단지 주변 불러오기" → 반경 bbox로 Overpass 쿼리 → GeoJSON 캐시
- 로딩 상태·출처·라이선스 표기 자동 삽입

### 6.3 직접 그리기 (펜 도구)
- 베지어 라인/폴리곤 수기 작성 (예정 노선·구획)
- 색상·두께·점선 스타일 지정 → 경로 B 익스포트 엔진과 공유

### 6.4 익스포트
- "PPT로 내보내기" → 하이브리드 슬라이드 생성
- 범례(레전드)·축척·출처 자동 포함

## 7. 단계별 구현 로드맵

| 단계 | 내용 | 산출물 | 규모 |
|------|------|--------|------|
| **M0 · PoC** | MapLibre + VWorld 타일 + 천안 샘플 단지 핀 | "지도가 뜬다" 확인 | 반나절 |
| **M1 · 반경+레이어** | turf 버퍼 동심원 + Overpass로 철도·도로 추출·렌더 + 레이어 토글 | 트레이싱 제거 시연 | 2~3일 |
| **M2 · 포인트 레이어** | 학교·병원·역 점 데이터 + 라벨 | 근린입지 지도 완성 | 1~2일 |
| **M3 · 이미지 익스포트(A)** | PNG → PptxGenJS 1장 자동 생성 + 범례 | PPT 산출 | 1~2일 |
| **M4 · 직접 그리기** | 베지어 펜 도구 (예정 노선) | 수기 보강 | 2~3일 |
| **M5 · 벡터 익스포트(B)** | 선/구획 custGeom 편집 가능 도형 | 핵심 R&D | 별도 덩어리 |
| **M6 · 광역입지 모드** | 시·도 범위 + 경계 폴리곤 | 보고서 광역 항목 | 1~2일 |

> M0~M3까지가 "실무 시간 절감" 1차 가치. M5(벡터 익스포트)는 R&D 성격이라 일정 분리.

## 7.5 API 키 보관 규약 (확정)

- 키는 프로젝트 루트의 **`keys.local.env`** 에 `KEY=VALUE` 형식으로 보관 (한 줄당 하나)
  ```
  MAPTILER_KEY=xxxxxxxxxxxx
  # 향후: VWORLD_KEY=..., NAVER_CLIENT_ID=... 등 추가
  ```
- **`.gitignore`에 등록되어 절대 커밋되지 않음** (`keys.local.env`, `*.local.env`, `.env*`)
- 앱은 런타임에 `fetch("keys.local.env")` 로 읽어 파싱 (빌드 없음 환경 대응) — `js/mapStudio.jsx`의 `loadKeys()`
- 키 부재 시 OSM 래스터로 자동 폴백 → 키 없이도 동작
