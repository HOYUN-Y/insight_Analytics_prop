====================================================================
0. Planning Studio (Business Understanding)
====================================================================

이 프로그램은 데이터 분석 프로그램이면서 동시에
분양대행사 기획 담당자의 업무 지원 도구이다.

따라서 데이터 수집 이전 단계인
Business Understanding(비즈니스 이해) 기능을 제공해야 한다.

목적:
- 프로젝트 목표 정의
- 분석 목적 정리
- 조사 범위 설정
- 가설 수립
- 연구 질문 정리
- 인사이트 축적
- 전략 수립

Planning Studio는 프로젝트 생성 시 가장 먼저 열리는 화면이다.

====================================================================
Planning Studio 설계 원칙
====================================================================

이 모듈은

JMP
Tableau
Power BI

가 아닌

Miro
FigJam
Notion
Obsidian

의 장점을 참고한다.

목표는

"데이터를 분석하는 프로그램"

이 아니라

"분양기획 업무 전 과정을 지원하는 Planning OS"

를 만드는 것이다.

Planning Studio는
프로젝트의 모든 데이터, 차트, 지도, 메모, 인사이트의
최상위 허브 역할을 수행해야 한다.


너는 로컬 웹 애플리케이션을 설계·개발하는 시니어 프로덕트 엔지니어다.

현재 프로젝트는 “insight Analytics Workbench”라는 로컬 브라우저 기반 데이터 분석 프로그램이다.
기존 방향은 Tableau식 시각화, JMP식 통계/EDA, Power BI식 대시보드, 데이터 전처리 기능을 결합한 개인용 분석 도구였으나, 이번에는 이를 “분양대행사 마케팅 기획 직무 전용 분석 프로그램”으로 재기획한다.

중요한 전제:
- 이 프로그램은 ML/DL 모델 개발이나 지속 운용을 목표로 하지 않는다.
- 분양대행사 기획 실무에서는 모델 셀렉션 이전 단계, 즉 데이터 수집, 정리, EDA, 시각화, 시장조사, 비교 분석, 보고서/제안서용 인사이트 정리가 핵심이다.
- 최종 인사이트 판단은 사람이 한다.
- 따라서 AutoML, 고급 예측모델, 딥러닝 기능은 제거하거나 후순위로 둔다.
- 대신 부동산 시장조사 보고서와 마케팅 제안서를 빠르게 만들 수 있도록 데이터 수집, 수기 입력, 전처리, 시각화, 비교분석, 리포트 산출 흐름을 강화한다.

목표 제품명:
- 가칭: Real Estate Planning Workbench
- 또는 insight Planning Workbench
- 로컬 환경의 웹 브라우저에서 실행
- 가능하면 기존 React + ECharts 기반 구조를 유지
- 데이터는 기본적으로 사용자의 로컬에만 저장
- 프로젝트 단위로 자료, 데이터셋, 차트, 메모, 보고서 초안을 관리

핵심 사용자:
- 분양대행사 마케팅 기획 담당자
- 부동산 데이터를 기반으로 시장조사 보고서, 입지 분석 자료, 분양 제안서, 마케팅 전략 자료를 작성하는 사람

핵심 업무 흐름:
1. 프로젝트 생성
2. 사업지/현장 정보 입력
3. 외부 데이터 수집 또는 수기 입력
4. 데이터 정리 및 전처리
5. 입지/시장/경쟁단지/수요/가격/분양조건 분석
6. 차트와 지도 시각화 생성
7. 사람이 해석 메모 작성
8. 보고서/제안서용 표·차트·요약 문구 export

필수 모듈:

1. Project 모드
- 프로젝트 단위 관리
- 프로젝트명, 사업지명, 주소, 시행/시공/분양 관련 정보 입력
- 사업 유형 선택: 아파트, 오피스텔, 지식산업센터, 상가, 생활숙박시설 등
- 분석 목적 선택: 시장조사, 제안서, 경쟁단지 비교, 가격 검토, 입지 분석 등
- 프로젝트별 데이터셋, 차트, 지도, 메모, 리포트 초안 저장
- 최근 프로젝트 목록
- 프로젝트 복제 기능

2. Data 모드
- CSV, XLSX, JSON import
- 직접 입력용 빈 시트 생성
- 엑셀처럼 행/열 추가, 삭제, 수정 가능
- 컬럼 타입 지정: 문자, 숫자, 날짜, 금액, 면적, 비율, 주소, 위도, 경도
- 자주 쓰는 부동산 데이터 템플릿 제공
  - 경쟁단지 조사표
  - 분양가 비교표
  - 입지시설 조사표
  - 교통환경 조사표
  - 인구/세대수 데이터
  - 청약경쟁률 데이터
  - 실거래가/매물가 비교표
  - 납부조건 비교표
  - 마케팅 채널 성과표
- 데이터셋별 설명 메모 작성
- 데이터셋 export: CSV, JSON, XLSX 가능하면 지원

3. Manual Collection 모드
- 사용자가 직접 조사한 데이터를 입력하는 전용 화면
- 빈 시트 기반 입력
- 행 단위 카드 입력 모드와 그리드 입력 모드 둘 다 제공
- 예시 입력 항목:
  - 단지명
  - 주소
  - 세대수
  - 입주년월
  - 평형
  - 전용면적
  - 공급면적
  - 분양가
  - 평당가
  - 실거래가
  - 매물가
  - 전세가
  - 청약경쟁률
  - 계약조건
  - 중도금 조건
  - 발코니 확장비
  - 주차대수
  - 역세권 여부
  - 학교/상권/병원/공원 거리
  - 비고
- 입력 중 자동 계산:
  - 평당가
  - 전용률
  - 연식
  - 가격 차이
  - 가격 증감률
  - 경쟁단지 대비 프리미엄/디스카운트
- 빠른 입력을 위한 드롭다운/자동완성/복사 붙여넣기 지원
- 출처 URL/캡처 메모/조사일자 필드 제공

4. API Data Connector 모드
- 공공데이터, 부동산 관련 API, 지도/주소 API 등을 연결할 수 있는 구조
- 실제 API 키는 사용자가 로컬에서 입력
- API Connector는 플러그인 방식으로 설계
- 초기 구현은 Mock Connector 또는 설정 가능한 Generic REST API Connector로 시작
- 기능:
  - API endpoint 등록
  - query parameter 설정
  - 응답 JSON preview
  - 필요한 필드만 선택
  - 테이블로 변환
  - 프로젝트 데이터셋으로 저장
- 향후 확장 가능한 Connector 예시:
  - 공공데이터포털
  - 국토교통부 실거래가
  - KOSIS 인구/세대
  - 한국부동산원
  - VWorld/주소 좌표 변환
  - 네이버/카카오 지역 검색 API

5. Clean / Preprocessing 모드
- 기존 Clean 기능을 유지하되, 기획 실무에 맞게 표현을 쉽게 변경
- 결측치 처리
- 중복 제거
- 컬럼명 정리
- 데이터 타입 변환
- 금액 단위 변환: 원, 만원, 억원
- 면적 단위 변환: ㎡, 평
- 날짜 파싱
- 주소 정제
- 숫자 문자열 정리
- 이상치 확인 및 제외
- 파생 컬럼 생성
- 수식 기반 컬럼 생성
- 표준화/정규화는 유지하되 고급 분석용이 아닌 비교지표 생성용으로 설명
- Undo/Redo 지원
- 전처리 과정은 Pipeline으로 기록

6. Real Estate Analysis 모드
- 분양 기획 실무에 맞는 분석 화면 제공
- 하위 탭:
  1) 시장 개요
  2) 가격 분석
  3) 경쟁단지 비교
  4) 입지 분석
  5) 수요/인구 분석
  6) 분양조건 비교
  7) 마케팅 포인트 정리

- 가격 분석:
  - 평균/중앙값/최소/최대
  - 평당가 비교
  - 면적대별 가격
  - 연식별 가격
  - 지역별 가격
  - 경쟁단지 대비 가격 차이
  - 가격 포지셔닝 매트릭스

- 경쟁단지 비교:
  - 단지별 세대수, 연식, 거리, 가격, 브랜드, 입지시설 비교
  - Radar Chart
  - Ranking Table
  - Positioning Map
  - Strength / Weakness 메모

- 입지 분석:
  - 교통, 교육, 상권, 자연환경, 업무지구 접근성 정리
  - 시설별 거리 테이블
  - 지도 기반 포인트 표시
  - 반경 분석: 500m, 1km, 3km 등
  - 사용자가 직접 시설을 추가할 수 있어야 함

- 수요/인구 분석:
  - 인구수
  - 세대수
  - 연령대
  - 소득 또는 구매력 지표가 있다면 시각화
  - 전입/전출
  - 가구 구성
  - 추세 차트

- 분양조건 비교:
  - 계약금
  - 중도금
  - 잔금
  - 중도금 무이자 여부
  - 발코니 확장비
  - 옵션비
  - 납부조건 타임라인
  - 경쟁 사업지 대비 유리/불리 포인트 정리

- 차트 커스터마이징:
  - 차트 제목/부제 수정
  - X축/Y축 필드 변경
  - 축 범위 수동 설정
  - 로그 스케일/선형 스케일 선택
  - 단위 표시: 원, 만원, 억원, 평, ㎡, %, 세대
  - 라벨 표시/숨김
  - 라벨 포맷 설정
  - 범례 위치 변경
  - 그리드라인 표시/숨김
  - 색상 팔레트 선택
  - 개별 시리즈 색상 커스터마이징
  - 강조 색상 지정
  - 평균선/기준선 추가
  - 목표값 라인 추가
  - 주석 annotation 추가
  - 차트 크기 조절
  - PNG/SVG export
  - 보고서용 투명 배경 export 가능하면 지원

13. AI Assistant / Ask Insight
- 실제 모델 예측이 아니라 분석 보조 역할
- 가능한 기능:
  - 현재 데이터셋 요약
  - 이상한 값 탐지
  - 어떤 차트를 그리면 좋을지 추천
  - 보고서 문구 초안 제안
  - 경쟁단지 비교 관점 제안
  - 추가로 수집해야 할 데이터 제안
- 중요한 제한:
  - AI가 최종 판단을 내리지 않게 한다.
  - “판단”보다 “검토 포인트 제안” 중심으로 작성한다.
  - 로컬 LLM 또는 외부 API 연동은 선택 사항으로 설계한다.

우선순위:
1순위:
- 프로젝트 관리
- CSV/XLSX/JSON import
- 빈 시트 수기 입력
- 부동산 조사 템플릿
- Clean/Preprocessing
- 강력한 Chart 커스터마이징
- Map 시각화
- Insight Memo
- Report Builder

2순위:
- API Connector
- 대시보드
- 자동 차트 추천
- 보고서 문구 초안
- 출처 관리

3순위:
- 고급 통계
- PCA
- ML/AutoML
- 예측모델
- 딥러닝

제거 또는 후순위 처리할 기능:
- AutoML 중심 화면
- Logistic Regression, Decision Tree, Random Forest 등 모델 확장
- Confusion Matrix, ROC/AUC 등 분류모델 평가 지표
- Neural Network
- 복잡한 모델 비교 이력
- 분양 기획 실무에 직접 쓰기 어려운 고급 알고리즘

개발 시 주의:
- 기존 프로그램의 로컬 퍼스트 구조를 유지한다.
- 기존 ECharts 기반 차트 시스템을 확장한다.
- 기존 Clean 모드의 전처리 기능은 유지하되 실무 용어로 재구성한다.
- 기존 Map 모드는 부동산 입지 분석 중심으로 재구성한다.
- 기존 Stats 모드는 “통계 분석”보다 “EDA / 분포 / 비교 / 요약” 중심으로 단순화한다.
- 모든 기능은 비전문가도 사용할 수 있게 설명 문구를 제공한다.
- 분석 결과마다 “이 수치가 의미하는 바를 사용자가 직접 메모하는 구조”를 제공한다.
- 데이터 규모는 로컬 브라우저 기준 1만~5만 행 정도를 현실적 기준으로 삼고, 큰 데이터에서는 경고를 표시한다.

최종 산출물:
- 기능 중심 PRD
- 화면별 기능 명세
- 데이터 모델 설계
- 모듈 구조 제안
- 기존 insight Analytics Workbench에서 유지/수정/제거할 기능 목록
- Claude Code가 바로 개발에 착수할 수 있는 단계별 구현 계획
